# GitHub Pages: опубликуй сайт за 10 минут

Бесплатный хостинг от GitHub для статичных сайтов. Не нужно платить за сервер, настраивать домен или знать DevOps. Достаточно аккаунта на GitHub и HTML-файла, сгенерированного нейросетью.

---

## Что можно опубликовать

- Лендинг для продукта или услуги
- Презентация (как эта)
- Портфолио
- Мини-сайт компании или мероприятия
- Инструкция или документация
- Страница для сбора контактов (статичная форма через сторонний сервис)

---

## Пошаговая инструкция

### 1. Зарегистрироваться на GitHub

Зайди на [github.com](https://github.com) → нажми **Sign up** → введи email, придумай логин и пароль → подтверди email.

Это бесплатно. Логин станет частью адреса твоего сайта: `твой-логин.github.io`.

---

### 2. Создать репозиторий

На главной странице нажми зелёную кнопку **New** (или плюс в шапке).

- **Repository name** — придумай название, например `my-site`. Название тоже войдёт в адрес сайта.
- Видимость — выбери **Public** (для GitHub Pages это обязательно на бесплатном тарифе).
- Нажми **Create repository**.

---

### 3. Сгенерировать сайт в нейросети

Открой [claude.ai](https://claude.ai) или [chatgpt.com](https://chatgpt.com) и напиши примерно такой запрос:

> Сделай одностраничный HTML-сайт о [твоя тема]. Один файл, без внешних зависимостей. Современный дизайн, тёмная тема.

Скопируй весь сгенерированный код и сохрани его в файл с названием **`index.html`** — именно так, не `site.html` и не `index.htm`. GitHub Pages ищет этот файл по умолчанию.

**Советы для промпта:**
- Опиши цель страницы и целевую аудиторию
- Укажи нужные блоки: заголовок, описание, контакты, кнопка CTA
- Попроси «без внешних зависимостей» — тогда файл будет полностью автономным
- Попроси добавить мета-теги для соцсетей, если хочешь нормальный превью при шаринге

---

### 4. Загрузить файл в репозиторий

Зайди в созданный репозиторий на GitHub.

- Нажми **Add file** → **Upload files**
- Перетащи `index.html` в область загрузки
- Внизу страницы нажми **Commit changes**

Если у сайта есть папка с картинками или CSS — загрузи их тоже, сохраняя структуру папок. Но если генерировал через нейросеть с условием «без зависимостей», всё уже внутри одного файла.

---

### 5. Включить GitHub Pages

В репозитории перейди во вкладку **Settings** → в левом меню найди раздел **Pages**.

- В поле **Branch** выбери `main`
- Папку оставь `/ (root)`
- Нажми **Save**

GitHub запустит сборку. Подождать 1–2 минуты.

---

### 6. Открыть сайт

Обновить страницу Settings → Pages. Появится зелёный баннер со ссылкой вида:

```
https://твой-логин.github.io/my-site
```

Эта ссылка работает у всех без регистрации. Можно сразу делиться.

---

## Ограничения

| Что | Подробности |
|-----|-------------|
| Только фронтенд | HTML, CSS, JavaScript — но не PHP, Python, Node.js |
| Нет базы данных | Нельзя хранить данные пользователей на сервере |
| Нельзя принимать оплату | Нет серверной логики для обработки платежей |
| Только публичные репо | На бесплатном тарифе — сайт виден всем |
| Лимит размера | До 1 ГБ на репозиторий, до 100 МБ на файл |

Если нужна форма обратной связи — используй [Formspree](https://formspree.io) или [Tally](https://tally.so): они принимают данные и шлют на email, а твой сайт остаётся статичным.

---

## Цена

Бесплатно для публичных репозиториев. Всегда.

Платный тариф GitHub Pro нужен только если хочешь сделать сайт из приватного репозитория.

---

## Обновить сайт

Просто загрузи новый файл поверх старого через **Add file → Upload files** → **Commit changes**. Изменения появятся через 1–2 минуты.

---

## Подключить свой домен (опционально)

Если есть домен (например, купленный на REG.RU или Cloudflare):

1. В DNS-настройках домена добавь CNAME-запись: `www` → `твой-логин.github.io`
2. В Settings → Pages → Custom domain введи свой домен
3. Поставь галочку **Enforce HTTPS**

GitHub сам выпустит SSL-сертификат через Let's Encrypt.
